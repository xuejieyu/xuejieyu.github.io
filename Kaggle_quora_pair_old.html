Cracking the Kaggle project-Quora question pair

This is the Kaggle project Quora Question Pairs competition: https://www.kaggle.com/c/quora-question-pairs. The goal for this challange is to compare two questions from quora, and identify if they're duplicate. This can come very useful when there are tons of questions being asked every day, and consolidate the answer for the convenience of users.

This aritile provide a comprehensive step-by-step code of solutions, from data exploration, handcrafted featureing, (including the discovery of magic features), to training the model by serveal machine learning agrolritms (random forest, deep learning, xgboost), and pick the winner.

Handcrafted featurings¶

In order to analyze the file, we need translate the text into nums, so the computer can understand and be able to calculate. There are tons of ways to do so. I will use something called TF-IDF (term-frequency-inverse-document-frequency). TF is to calculate how many terms are in common from both questions, IDF is the inverse document frequency, which means that we weigh the terms by how uncommon they are, meaning that we care more about rare words existing in both questions than common one.

This makes sense, as for example we care more about whether the word "python" appears in both than the words "and" or "the" - as uncommon words will be more indicative of the content. Also, we try to see if stop words like 'an' 'the' would play a role in it, so we will build features based on both.

In the last but not least, we want put bag of words together as normally phases carry out the meaning different from each seperate words.
